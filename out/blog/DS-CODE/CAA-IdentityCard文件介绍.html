CAA-IdentityCard文件介绍

2023-08-10
编辑本页

IdentityCard文件介绍：

IdentityCard.h文件相当于“附加包含目录”,你可能会被用来设置在一个典型的visual studio项目。它是一个目录列表,在CATIA实体的代码驻留它定义它们的属性和行为。当您使用一个类在CAA项目包含头文件,但是编译器需要知道在哪里找到这个头文件，mkmk编译器仍然需要知道它在哪里)。
IdentityCard.h文件是以以下格式组织在一起：


'''CAA Wizard Edition Zone'''
// DO NOT EDIT :: THE CAA2 WIZARDS WILL ADD CODE HERE
AddPrereqComponent("System",Protected);
AddPrereqComponent("CATAnalysisInterfaces",Protected);
AddPrereqComponent("DialogEngine",Protected);
AddPrereqComponent("Mathematics",Protected);
AddPrereqComponent("Dialog",Protected);
// END WIZARD EDITION ZONE

*This code is automatically generated by the CAA Wizards and should not be adjusted or changed.

'''User Input Area'''
AddPrereqComponent("ApplicationFrame",Public);
AddPrereqComponent("GSMInterfaces",Public);
AddPrereqComponent("SketcherInterfaces",Public);
AddPrereqComponent("MecModInterfaces",Public);
AddPrereqComponent("ObjectSpecsModeler",Public);
AddPrereqComponent("GeometricObjects",Public);
AddPrereqComponent("ObjectModelerBase",Public);

... 每次你用一个新的类对象在您的代码中,您必须确保编译器知道头文件定义在哪里。您必须添加头文件在文件夹驻留此列表。这可以通过如下:
　　:#打开头文件在Visual Studio关联到一个特定的对象通过强调类对象的名称和按ctrl + T。
　　:#打开Windows资源管理器显示了当前打开的文件的位置在Visual Studio通过按ctrl + Shift + x。
:#在显示的文件夹路径复制文件夹名称,是同一级别。

:# Add”这个词作为函数的第一个输入“AddPrereqComponent”来产生的IdentityCard.h文件:

　　AddPrereqComponent("Mathematics",Public);

注:“Public”和“Protected”并没有被很好的理解在这一点上。VC++向导通常的自动添加“受保护的”,然而,你应该使用“Public”当你添加的链接目录。
　　注:‘记住只是因为你有包括在IdentityCard。h文件，并不意味着个人dll可以看到,他们需要的代码。你必须还包括一个模块在Imakefile使代码可用到每个dll。

CAA RADE Imakefile
Imakefile.mk就是在IdentityCard.h文件包含的framework中用到的特定的dll模块被关联到Imakefile。
'''Wizard Edition Zone'''

DO NOT EDIT :: THE CAA2 WIZARDS WILL ADD CODE HERE

WIZARD_LINK_MODULES = JS0GROUP \
JS0FM JS0GROUP CATAnalysisInterface CATAnalysisInterfacesUUID \
DI0PANV2 CATMathematics CATDialogEngine

END WIZARD EDITION ZONE

:*This section is generated automatically by CAA RADE wizards. We really don't need to worry about this section.

'''Link_With Zone'''

LINK_WITH = $(WIZARD_LINK_MODULES) \
CATApplicationFrame \
CATMecModInterfaces \
CATSketcherInterfaces \
CATGitInterfaces \
CATObjectSpecsModeler \
CATDialogEngine \
CATMathematics \

“$(WIZARD_LINK_MODULES)“变量定义在向导版区,不应改变。
　　每个类的对象可以把它添加到你的代码,你必须添加一个关键字模块名到Imakefile Link_With区域链接类的代码到你的dll。
　　反斜杠 \ 只不过是一个行连续字符
　　如果你想让Imakefile中的注释,您必须使用“#”。
　　最好的办法是每一次使用一个新类对象在您的代码中,添加它所在的模块名在Imakefile中。如果你不做这个的时候,你将无法找出哪些类你忘记做这件事。
